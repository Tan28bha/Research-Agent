WHAT THE DATA SAYS

The patterns matter more than the 100 rows.

The research was designed to identify ecosystem-level patterns across authentication, developer access, API breadth, MCP availability, and integration feasibility.

01

Authentication is rarely the hardest problem.

Established authentication patterns are common. Credential availability and developer access restrictions create greater integration friction.

02

Public documentation does not mean public access.

Several ecosystems document APIs openly while restricting usable credentials behind paid plans, approval processes, or enterprise relationships.

03

Developer platforms offer the clearest easy wins.

API-first products generally combine broader surfaces, clearer documentation, and lower credential friction.

04

MCP coverage remains fragmented.

Existing MCP implementations reduce integration effort for some products, but direct API tooling remains necessary across much of the landscape.

HONEST FAILURES

Where the agent needed help.

The system was intentionally designed to surface uncertainty rather than hide it.

FAILURE MODE 01

Documentation ≠ credential access

Some applications expose public documentation while restricting actual developer credentials. First-pass research occasionally conflated the two.

Correction

Access-model verification was separated from API-documentation discovery.

FAILURE MODE 02

MCP evidence can be ambiguous

Search results sometimes surfaced AI integrations, plugins, or community projects that were not actual MCP implementations.

Correction

MCP classification required evidence of an actual implementation.

FAILURE MODE 03

Weak evidence created false confidence

A valid URL does not automatically support the associated research claim.

Correction

Claim-level evidence verification and targeted retries were added.

METHODOLOGY

Built for scale, designed for uncertainty.

The research set contains 100 applications across ten categories. Every application was processed independently to reduce cross-app contamination.

A discovery pass searched for official developer documentation and access information. A structured research pass converted the findings into a strict schema covering authentication, access, API breadth, MCP availability, buildability, blockers, evidence, and confidence.

A second evidence-checking pass challenged the first result. Low-confidence or contradicted findings triggered targeted retries before final verification.

Finally, a stratified sample across all ten categories was manually checked against real documentation.
